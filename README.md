# Reservation Client

Compose Multiplatform клиент универсальной системы бронирования временных слотов.

Приложение предоставляет единый пользовательский интерфейс для работы с сервисом бронирования и поддерживает все основные сценарии как для обычных пользователей, так и для администраторов услуг.

## Поддерживаемые платформы

* Android
* iOS
* Desktop
* Web (Compose Multiplatform)

## Технологии

* Kotlin Multiplatform
* Compose Multiplatform
* Koin
* Ktor Client
* Coroutines
* Kotlinx Serialization
* Google OAuth 2.0

## Возможности

### Авторизация

* Вход через Google OAuth 2.0;
* Автоматическое восстановление пользовательской сессии;
* Хранение access и refresh токенов;
* Выход из аккаунта.

### Пользовательские возможности

Пользователь может:

* просматривать список доступных услуг;
* просматривать детальную информацию об услуге;
* просматривать доступные временные слоты;
* создавать бронирования;
* отменять собственные записи;
* просматривать историю своих бронирований.

### Администрирование услуг

Администратор может:

* создавать услуги;
* редактировать услуги;
* управлять администраторами услуги;
* назначать нескольких администраторов;
* просматривать все бронирования услуги;
* отменять бронирования клиентов с указанием причины.

### Управление расписанием

Поддерживаются:

* постоянные правила доступности;
* индивидуальные правила доступности;
* периоды недоступности;
* управление буфером между записями.

### Фильтрация

Администратор может фильтровать бронирования:

* по пользователю;
* по дате;
* с возможностью сброса любого фильтра.

## Запуск проекта

### Предварительная настройка

Перед запуском приложения необходимо указать адрес backend-сервера.

В классе `ApiService` измените значение `baseUrl`:

```kotlin
abstract class ApiService(private val apiSessionManager: AuthSessionManager) {
    companion object {
        val baseUrl: String = "https://your-backend-url"
    }
}
```

Например, при использовании локального сервера:

```kotlin
val baseUrl = "http://localhost:8080"
```

или:

```kotlin
val baseUrl = "https://your-production-domain"
```

> Backend должен быть предварительно запущен и доступен по указанному адресу.

### Android

Для запуска приложения на устройстве или эмуляторе:

* открыть проект в Android Studio;
* запустить импортированную Android-конфигурацию.

Сборка APK:

```bash
./gradlew :androidApp:assembleDebug
```

Готовый APK:

```text
androidApp/build/outputs/apk/debug/androidApp-debug.apk
```

---

### Desktop

Запуск desktop-приложения:

```bash
./gradlew :desktopApp:run
```

Запуск с поддержкой hot reload:

```bash
./gradlew :desktopApp:hotRun --auto
```

---

### iOS

Для запуска приложения:

* открыть `iosApp/iosApp.xcodeproj` в Xcode;
* выбрать устройство или симулятор;
* выполнить стандартный запуск.

Либо использовать Kotlin Multiplatform Mobile Plugin для Android Studio.

### Web (JS)


### Web (WASM)

### Web Distribution

Сборка production-версии web-приложения:

```bash
./gradlew :webApp:composeCompatibilityBrowserDistribution
```

Готовая сборка будет находиться в директории:

```text
webApp/build/dist/composeWebCompatibility/productionExecutable
```

Содержимое данной директории может быть размещено на любом статическом веб-хостинге.

## Используемый Backend

Клиент предназначен для работы с серверной частью Reservation Backend.

Для корректной работы должны быть доступны следующие возможности сервера:

* Google OAuth 2.0;

* управление услугами;
* бронирования;
* правила доступности;
* администрирование услуг.
